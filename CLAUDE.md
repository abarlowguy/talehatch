# TaleHatch — Project Constitution

## What this is
AI story-building app for kids. Kids answer guided prompts, Claude generates immersive chapters with illustrations. Live at talehatch.com.

## Tech Stack
- **Framework:** Next.js (App Router for pages, Pages Router for API routes)
- **Runtime:** Bun + Turbopack
- **DB:** Neon Postgres via `@neondatabase/serverless`
- **AI:** `claude-haiku-4-5-20251001` (Q&A/guided prompts), `claude-sonnet-4-6` (chapter generation + editing)
- **Images:** Pollinations AI (`image.pollinations.ai`, `model=turbo`) — proxied via `/api/image-proxy`
- **Hosting:** Vercel
- **Styling:** Tailwind CSS

## Key Files
- `app/page.tsx` — entire client app: landing, age-select, guided Q&A, building state, chapter view. Contains `ChapterBody` and `ImageLoader` components.
- `components/StoryPicker.tsx` — email login, story hub (list, resume, delete stories)
- `lib/storyBuilder.ts` — typed fetch wrappers for all API routes
- `lib/prompts.ts` — guided prompts, age range config (`AgeRange`, `AGE_RANGE_CONFIG`)
- `lib/db.ts` — all Neon Postgres queries (initDb, getStory, createStory, updateStory, deleteStory)
- `lib/entityExtractor.ts` — noun extraction for entity tracking across chapters
- `pages/api/chapter.ts` — chapter generation (Anthropic → title + chapter + cliffhanger + imageUrls + storyTitle)
- `pages/api/generate.ts` — guided Q&A segment generation
- `pages/api/edit-chapter.ts` — chapter editing via prompt
- `pages/api/image-proxy.ts` — proxies Pollinations requests (browser fetch() is blocked by 403; proxy bypasses it)
- `pages/api/stories/index.ts` — GET/POST stories by email
- `pages/api/stories/[id].ts` — GET/PUT/DELETE a single story

## Architecture Decisions
- **App Router pages + Pages Router API:** Next.js hybrid — pages in `app/`, APIs in `pages/api/`. Don't move API routes to `app/api/`.
- **Sequential image loading:** `SlotState` queue in `ChapterBody` — only one `ImageLoader` fires at a time. Prevents 429s from Pollinations.
- **Image proxy mandatory:** Pollinations blocks browser `fetch()` (403 on `Sec-Fetch-Mode: cors`). All image loads go through `/api/image-proxy`.
- **`model=turbo` for images:** `model=flux` takes 30–60s; turbo is 1–3s. Don't switch back.
- **`nologo=true` removed:** Requires Pollinations premium — causes 403 for free tier.
- **Email normalization:** `email.toLowerCase()` at API layer + `LOWER(user_email)` in SQL. Both layers always.
- **Auto-save is fire-and-forget:** `autoSave()` never blocks the UI. Errors are swallowed silently.
- **`storyTitle` vs `chapterTitle`:** `storyTitle` is generated by Claude on chapter 1 and persists across chapters. `chapterTitle` is per-chapter. `autoSave` uses `storyTitle` if set, falls back to `buildStoryTitle()`.

## Environment Variables
- `DATABASE_URL` — Neon Postgres connection string
- `ANTHROPIC_API_KEY` — Anthropic API key

## Dev
```bash
bun dev          # start on port 3000
kill $(lsof -ti:3000)   # kill dev server if Turbopack gets stuck after a parse error
```

## Off-limits
- Don't modify the DB schema in `initDb()` without considering existing rows in production.
- `app/test-chapter/page.tsx` — delete this before any public launch.
- Don't add `nologo=true` to Pollinations URLs.
